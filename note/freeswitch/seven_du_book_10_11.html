  • 使用外部抓包工具

    1. tcpdump

      // -n、-q：不进行域名翻译(使用数字域名)及减少输出内容
// -s 0：不限制包长，争取抓最大的长度
// -A：以ASCII方式输出
// -v：显示的详细程度，v越多越详细
// -i：使用指定的网卡
// port 5060：过滤器，只抓取5060上的包
tcpdump -nq -s 0 -A -vvv -i eth0 port 5060

// -w：将结果写入文件
tcpdump -nq -s 0 -A -vvv -i eth0 -w /tmp/dump.pcap port 5060

// 只抓取udp包
tcpdump -nq -s 0 -A -vvv -i eth0 -w /tmp/dummp.pcap udp

// 只抓取某一用户的包
// 查询用户使用的ip和端口
sofia status profile internal reg 1002
// ip:port抓包
tcpdump -i eth0 -s 0 -A host 192.168.7.5 and port 65272

// 对rtp包抓包，freeswitch默认的端口范围是16384 ~ 32768
tcpdump -i eth0 -w /tmp/sip-rtp.pcap "udp and (port 5060 or port 5080 or portrange 16384-32768)"

    2. tshark(wireshark的命令行工具yum install -y wireshark)

      能认识很多协议

      tshark

tshark -i eth0

tshark -i eth0 -w /tmp/dump.pcap

tshark host 192.168.1.144 and port 5060

    3. ngrep

      类似于grep，可以提供类似于freeswitch内部抓包的输出方式

      // -p：不使用混杂模式(在一台主机上接收局域网所有的包)
// -q: 使用安静模式，仅输出包头和相关包的载荷
// -W：显示的输出方式
// port 5060是一个过滤器
ngrep -p -q -W byline port 5060

// 指定ip和端口号
ngrep -p -q -W byline host 192.168.1.9 and port 5060

// 对内容进行过滤
ngrep -p -q '139xxxxxxxx' -W byline port 5060

    4. pcapsipdump

      将不同通话IP包存到不同的文件里，以便后续分析

      pcapsipdump -i eth0 -d /tmp/sipdump/

  • originate 命令实例解析

    <call url> <exten>|&<application_name>(<app_args>) [<dialplan>] [<context>] [<cid_name>] [<cid_num>] [<timeout_sec>]
    • 使用格式和参数
      freeswitch> originate user/1000 &echo
      1. user/1000是呼叫字符串，格式“类型/参数/参数”
      2. [系统提供多少channel类型]
      3. 每个Endpoint都会提供相应的呼叫字符串
      4. 每一种呼叫字符串的类型都属于一个Endpoint Interface
      5. 其中一些类型(如：user、group)看起来像虚拟的Endpoint Interface，最终会 解析底层Sofia Endpoint Interface
      6. 同振：一个接听了，另一个挂断
        originate user/1000,user/1001 &echo
      7. 顺振：第一个呼叫失败则呼叫第二个
        originate user/1000|user/1001 &echo
    • 转入dialplan
      1. exten: 可以认为是一个分机号
        // 两条命令基本一样
originate user/1000 9196
originate user/1000 &echo
      2. dialplan: 默认是XML，下面使用inline
        originate user/1000 echo inline
      3. context: 对于dialplan为inline的，它会忽略context，而对于XML确实有效的
        originate user/1000 1001 XML public
    • 更改主叫号码(Caller ID Number)
      1. 主叫名称(cid_name)
      2. 主叫号码(cid_number)
        // 有空格时用单引号引起来
originate user/1000 &echo XML default 'Seven Du' 7777

    • 处理超时呼叫

      不是对方不接电话超时的秒数，而是在对方收到invite消息后，不回复100trying的秒数

      // 192.168.100.100并不存在
originate sofia/internal/1000@192.168.100.100 &echo XML default 
'Seven Du' 777 10

      2. 开启另一个fs_cli

    • 使用通道变量
      1. 通道变量是加在呼叫字符串上的
        // 改变主叫名称和号码
originate {origination_caller_id_name='Seven Du',
origination_caller_id_number=777}user/1000 &echo
        通道变量使用逗号(,)分隔，所以通道变量中需要用到','的需要转义
        1. 使用'\'进行转义
          originate {absolute_codec_string='G729\,PCMU'}user/1000 &echo
        2. 使用^^:声明接下来的分隔符使用:
          originate {absolute_codec_string='^^:G729:PCMU'}user/1000 &echo
      2. 将不同的通道变量写到不同的大括号内
        originate {origination_caller_id_name='Seven Du'}
{origination_caller_id_number=777}user/1000 &echo
      3. 局部通道变量
        // leg_timeout只作用于靠近它的那条腿
originate {var=1}[leg_timeout=10]user/1000|
[leg_timeout=20]sofia/gateway/test/13675017141 &echo

    • Early Media 对呼叫的影响

      originate命令在收到媒体指示就返回，如SIP中的183或200消息。
      但是在电话自动催话费应用，直到用户接听才对其播放，或进入IVR；这时如果不忽略 早期媒体，播放完了催费声，用户还没接听就会挂断。使用忽略早期媒体解决：

      originate {ignore_early_media=true}sofia/gateway/test/13800000000
&playback(/tmp/test.wav)

    • bridge也使用originate

      originate user/1000 &bridge(user/1001)

originate user/1000
originate(NULL, new_session, "user/1000")
// 主被叫号码都是相同，使用0000000作为主叫号码
&bridge(user/1001)
originate(session_1000, new_session, "user/1001")
// 1000接听了，然后去桥接1001，使双方建立通话

    • bridge中的Early Media

      在上面的场景，由于1000在接听，1001无法给1000发送180消息，因而1000在1001振铃 期间听不到任何声音。让1000听到假的回铃音

      originate {transfer_ringback=local_stream://moh}user/1000 &bridge(user/1001)

      有时1000也要等很久时间才能收到1001的180消息，立即播放假回铃音

      originate {instant_ringback=true}{transfer_ringback=local_stream://moh}
user/1000 &bridge(user/1001)
    • bridge中的主叫号码
      // 1000上显示7777，1001显示8888
originate {origition_caller_id_number=7777}user/1000 &bridge(
{origination_caller_id_number=8888}user/1001)
// effective_caller_id_number会影响b-leg
originate {origition_caller_id_number=7777}{effective_caller_id_number=8888}
user/1000 &bridge(user/1001)

    • 录音相关的通道变量
      1. RECORD_STEREO:立体声
      2. RECORD_HANGUP_ON_ERROR
      3. RECORD_WRITE_ONLY
      4. RECORD_READ_ONLY
      5. RECORD_STEREO_SWAP：通立体声，但将左右声道切换
      6. RECORD_ANSWER_REQ:应答后才录音，即使执行了record_session也一样，不录早期媒体
      7. RECORD_BRIDGE_REQ:桥接完才开始录音
      8. RECORD_APPEND:不覆盖录音，追加录音
      9. record_sample_rate:在录音时指定采样pinl
      10. enable_file_write_buffering
      11. RECORD_MIN_SEC
      12. RECORD_INITIAL_TIMEOUT_MS：录音开始检测不到的超时秒数
      13. RECORD_FINAL_TIMEOUT_MS：录音过程检测不到的超时秒数
      14. RECORD_SILENCE_THRESHOLD:静音能量阈值

    • 原生格式：不指定文件后缀，则使用原生格式录音，节省系统资源

      原生格式不支持混音，只能将双向声道录到两个不同的文件(-in,-out)
      原生格式不包含录音文件的元信息（编码、采样率、声道等），需要制定

      // 播放
play -e u-law -r 8000 -t raw test-in.PCMU
// 转成wav文件
sox -e u-law -r 8000 -t raw test-in.PCMU test-in.wav
sox -e u-law -r 8000 -t raw test-in.PCMU test-out.wav

// 混音
sox test-in.wav test-out.wav -m test.wav
// 立体声
sox test-in.wav test-out.wav -M test.wav

    1000拨打#57#取消呼叫前转
    <!--conf/dialplan/default.xml-->
<extension name="call_forward_unconditional_set">
  <condition field="destination_number" expression="^\*57\*([^#]+)#?$">
    <!--oper/域/key/value-->
    <action application="hash" data="insert/${domain_name}-cfwdu/${username}/$1"/>
    <action application="answer"/>
    <action application="playback" data="/home/voices/rings/common/job_number_prefix.wav"/>
  </condition>
</extension>
<extension name="call_forward_unconditional_unset">
  <condition field="destination_number" expression="^#57#?$">
    <action application="hash" data="delete/${domain_name}-cfwdu/${username}/$1"/>
    <action application="answer"/>
    <action application="playback" data="/home/voices/rings/common/job_number_suffix.wav"/>
  </condition>
</extension>
<extension name="call_forward_unconditional_check" continue="true">
